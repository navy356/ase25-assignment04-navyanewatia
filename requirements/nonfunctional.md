# Non-Functional Requirements
- As a customer, I want the platform to be responsive and lightweight so that I can browse easily and without delays.
- As a customer, I want the platform to be intuitive to use so that I can order ingredients without confusion or frustration.
- As a registered customer, I want the platform to not lose order details if the system crashes so my orders remain accurate and trustworthy.
- As a developer, I want the platform to be easy to maintain and extend with new meal types or dietary rules.
- As an admin, I want the platfor to not lose order and meal data if it crashes so that the platform can be managed reliably.
